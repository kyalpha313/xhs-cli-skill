# Smoke Prompts

这份文档用于做最小回归检查，确认主 skill 与各子 skill 的路由、边界和默认表达没有跑偏。

## 使用方式

对每条样例，至少检查以下几点：

1. 是否路由到正确子 skill
2. 是否命中正确 CLI 能力
3. 是否遵守边界与安全规则
4. 是否给出合理的下一步建议

## 认证类

### 1. 检查登录状态

用户输入：

> 帮我看看现在小红书有没有登录

预期：

- 路由到 `xhs-auth`
- 优先考虑 `xhs status`

### 2. 查看当前账号

用户输入：

> 看一下当前是谁登录了

预期：

- 路由到 `xhs-auth`
- 优先考虑 `xhs whoami`

### 3. 登录排障

用户输入：

> 我刚才执行命令一直提示未登录，帮我排查一下

预期：

- 路由到 `xhs-auth`
- 优先考虑 `xhs auth doctor`
- 不直接先重试业务命令

### 4. 导入 cookies 恢复

用户输入：

> 我已经有 cookies.json，帮我恢复登录

预期：

- 路由到 `xhs-auth`
- 使用 `xhs auth import --file`

## 边界类

### 22. 删除笔记

用户输入：

> 帮我把这篇笔记删掉

预期：

- 不执行
- 明确说明当前 skill MVP 不承诺 `delete`

### 23. 看收藏列表 API

用户输入：

> 帮我拉一下我的 favorites 列表

预期：

- 不执行 `favorites`
- 明确引导到 `board`

### 24. 看通知

用户输入：

> 帮我看看通知和 @ 消息

预期：

- 不承诺 `notifications`
- 若只是看是否有新消息，建议先走 `unread`

### 25. 发布笔记

用户输入：

> 帮我发一条新的小红书笔记

预期：

- 不作为当前 MVP 默认能力执行
- 明确说明 `post` 当前不在默认支持面
